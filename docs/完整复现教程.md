# Chat Emotion 完整复现与使用教程

本文面向第一次接触本项目的使用者，目标是在一台新的 Windows 电脑上复现原作者电脑中的文本、Excel 批量文本、语音文件、图像文件和摄像头实时情感识别功能。

## 0. 开始前必须知道的事

仅克隆 GitHub 仓库还不能运行全部识别功能。仓库包含源码、测试、实验报告、轻量文本数据和科研绘图，但出于文件体积与第三方许可考虑，不包含以下内容：

1. 约 1.45 GB 的运行时模型权重；
2. RAF-DB、TESS、CREMA-D、EmoDB、OCEMOTION 等第三方原始数据；
3. 作者电脑上的 `.venv`；
4. `dist/EmotionRecognition.exe` 和训练缓存。

对普通使用者而言，**运行推理只需要源码、Python 环境和运行时模型包，不需要下载训练数据集**。若要从头训练并复现实验结果，才需要自行申请或下载数据集。

## 1. 可复现的功能范围

完整安装后可使用：

- 中文和英文单条文本七分类；
- Excel 文本批量识别；
- WAV、MP3 语音情感识别；
- PNG、JPG、JPEG、BMP、WebP 人脸图像识别；
- 默认摄像头实时人脸检测与情感识别；
- 七类情感概率可视化；
- 训练、评估、ONNX 导出与科研绘图脚本；
- PyInstaller 图像/语音演示版打包。

统一标签为：

```text
anger, disgust, fear, joy, sadness, surprise, neutral
```

## 2. 推荐硬件与软件

### 2.1 最低建议

- Windows 10 或 Windows 11，64 位；
- Python 3.10–3.12，64 位（推荐 3.11）；
- 8 GB 内存，推荐 16 GB；
- 至少 10 GB 可用磁盘空间；
- 普通 CPU 可运行，NVIDIA GPU 不是必需；
- 摄像头功能需要系统可识别的摄像头。

模型训练远比推理耗费资源。若要重训 XLM-R、RAF-DB 或 WavLM 相关模型，建议使用 NVIDIA GPU 和更大的磁盘空间。

### 2.2 安装基础软件

需要安装：

- Git；
- Python 3.10–3.12；
- 可选：GitHub Desktop、Visual Studio Code；
- 若 ONNX Runtime 报 DLL 错误，安装 Microsoft Visual C++ 2015–2022 x64 Runtime。

确认命令可用：

```powershell
git --version
python --version
```

Python 版本应显示 3.10.x、3.11.x 或 3.12.x，推荐 3.11.x。

确认目录中存在：

## 4. 创建独立 Python 环境

不要复制作者电脑的 `.venv`。虚拟环境包含绝对路径和本机二进制信息，跨电脑复制容易损坏。应在新电脑重新创建：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

如果 PowerShell 提示脚本执行被禁止，可仅对当前终端临时放行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

也可以始终不激活环境，直接使用：

```powershell
.\.venv\Scripts\python.exe app.py
```

### 4.1 验证依赖

```powershell
.\.venv\Scripts\python.exe -c "import PyQt5, torch, transformers, cv2, onnxruntime, pandas, miniaudio; print('dependencies OK')"
```

## 5. 准备运行时模型包

### 5.1 最小模型清单

要使用全部推理功能，项目目录必须具有以下结构：

```text
models/
├── text/
│   ├── config.json
│   ├── model.safetensors          # 或 pytorch_model.bin
│   ├── tokenizer.json
│   └── tokenizer_config.json
├── image/
│   ├── face_detection_yunet_2023mar.onnx
│   └── rafdb_se_resnet18/
│       └── rafdb_emotion.onnx
└── speech/
    ├── wavlm_simsan_encoder.onnx
    └── wavlm_simsan_head.joblib
```

可选文件：

- `models/image/rafdb_se_resnet18/metrics.json`：图像实验指标；
- `models/speech/wavlm_simsan_fixed_test_metrics.json`：语音实验指标；
- `models/speech/speech_model.joblib`：WavLM 模型不可用时的传统 MFCC-SVM 回退模型。

不要只创建空目录。每个模型文件都必须完整复制。

### 5.2 模型包从哪里获得

模型是本项目训练结果，不会由 `git clone` 自动下载。分享者需要采用下面一种方式提供：

1. 在 GitHub Release 中单独附加模型压缩包；
2. 使用 Git LFS 管理获准公开的权重；
3. 使用学校网盘、云盘或对象存储，并在 README 中给出下载地址和 SHA-256；
4. 接收者按训练教程自行生成。

分享模型前必须确认基础模型和训练数据许可允许分发派生权重。若许可不明确，优先提供训练脚本和由接收者自行生成权重的流程。

### 5.3 分享者制作最小运行时模型包

在原作者项目根目录运行：

```powershell
$bundle = "release-assets\runtime-models"
New-Item -ItemType Directory -Force "$bundle\models\text" | Out-Null
New-Item -ItemType Directory -Force "$bundle\models\image\rafdb_se_resnet18" | Out-Null
New-Item -ItemType Directory -Force "$bundle\models\speech" | Out-Null

Copy-Item models\text\config.json,models\text\model.safetensors,models\text\tokenizer.json,models\text\tokenizer_config.json "$bundle\models\text"
Copy-Item models\image\face_detection_yunet_2023mar.onnx "$bundle\models\image"
Copy-Item models\image\rafdb_se_resnet18\rafdb_emotion.onnx "$bundle\models\image\rafdb_se_resnet18"
Copy-Item models\speech\wavlm_simsan_encoder.onnx,models\speech\wavlm_simsan_head.joblib "$bundle\models\speech"

Compress-Archive -Path "$bundle\models" -DestinationPath "release-assets\chat-emotion-runtime-models.zip" -Force
Get-FileHash "release-assets\chat-emotion-runtime-models.zip" -Algorithm SHA256
```

若文本权重名为 `pytorch_model.bin`，将命令中的 `model.safetensors` 替换为该文件。

分享时同时提供压缩包和 SHA-256。接收者可校验：

```powershell
Get-FileHash .\chat-emotion-runtime-models.zip -Algorithm SHA256
```

### 5.4 接收者安装模型包

把压缩包放到项目根目录，解压：

```powershell
Expand-Archive .\chat-emotion-runtime-models.zip -DestinationPath . -Force
```

特别检查不要出现错误的双层目录：

```text
错误：chat_emotion/models/models/text/config.json
正确：chat_emotion/models/text/config.json
```

## 7. 启动桌面应用

```powershell
.\.venv\Scripts\python.exe app.py
```

首次文本预测会加载约 1 GB 的 XLM-R 权重，CPU 电脑可能等待数秒到几十秒。窗口没有立即返回结果不一定是故障，请观察任务状态并等待模型完成加载。

## 8. 各功能操作方法

### 8.1 单条文本识别

1. 打开“文本识别”页；
2. 输入中文或英文；
3. 点击“开始识别”；
4. 查看预测情感、置信度和七类概率。

文本模型必须完整包含七个标准标签。若 `config.json` 的 `id2label` 不匹配，程序会明确报错，不会生成随机结果。

### 8.2 Excel 批量文本识别

输入文件必须是 `.xlsx` 或 `.xls`，并至少包含以下任一列名：

- `text`
- `content`
- `文本`
- `内容`

示例：

| text |
| --- |
| 今天终于完成项目了，我非常开心。 |
| I am worried about the result. |

操作：

1. 点击“选择文件”；
2. 选择输出位置；
3. 点击“开始批量识别”。

输出表会新增：

- `预测情绪`；
- `置信度`；
- `错误信息`。

### 8.3 语音文件识别

1. 切换到“语音识别”；
2. 选择 WAV 或 MP3；
3. 点击“开始识别”。

当前界面支持选择已有音频文件，不包含麦克风录音按钮。模型读取音频开头最多约 4 秒，因此建议：

- 使用单人、清晰、无背景音乐的短语音；
- 避免长时间静音；
- 优先使用 WAV；
- MP3 无法读取时，先转成 16 kHz 单声道 WAV。

### 8.4 图像文件识别

1. 切换到“图像识别”；
2. 点击“导入图像”；
3. 选择包含清晰人脸的 PNG/JPG/JPEG/BMP/WebP；
4. 点击“识别图像”。

建议正脸、无遮挡、光照均匀且人脸占画面适中。多人图像会检测多张脸，文件识别结果以面积最大的人脸为主。

### 8.5 摄像头实时识别

1. 在“图像识别”页点击“启动摄像头”；
2. 允许 Windows 摄像头权限；
3. 保持脸部在画面内；
4. 点击“停止摄像头”或切换到其他标签页结束采集。

程序默认打开摄像头索引 0，目标分辨率为 640 × 480，每 100 ms 更新一次。

## 12. 关于 EXE 打包

若目标是复现全部功能，优先使用第 7 节的源码运行方式。

当前 `emotion_app.spec` 面向较轻量的图像/语音演示版，显式排除了 PyTorch、Transformers 和文本模型权重。因此直接执行：

```powershell
pyinstaller --noconfirm emotion_app.spec
```

不能保证得到包含完整文本模型的单文件 EXE。若要发布三模态完整 EXE，需要把 PyTorch、Transformers、分词器和 `models/text` 加入打包配置，最终文件会非常大，启动也更慢。

## 13. 常见故障

### 13.1 `ModuleNotFoundError`

确认使用的是项目虚拟环境：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe app.py
```

### 13.2 界面显示“未找到模型”

检查路径：

```powershell
Get-ChildItem models -Recurse -File
.\.venv\Scripts\python.exe scripts\verify_installation.py
```

最常见原因是压缩包多解压了一层目录。

### 13.3 文本模型加载失败

确认 `models/text` 同时包含权重、`config.json` 和 tokenizer 文件。程序采用 `local_files_only=True`，不会在运行时自动联网下载缺失文件。

内存不足时关闭其他大型程序，并先使用 CPU。不要删除模型目录中的配置文件。

### 13.4 `onnxruntime` 或 DLL 加载失败

- 确认 Python 和依赖均为 64 位；
- 安装 Microsoft Visual C++ 2015–2022 x64 Runtime；
- 重新安装：

```powershell
pip uninstall onnxruntime -y
pip install "onnxruntime>=1.27,<2"
```

### 13.5 摄像头打不开

- Windows 设置 → 隐私和安全性 → 摄像头，允许桌面应用访问；
- 关闭微信、会议软件和浏览器中占用摄像头的页面；
- 确认设备管理器中摄像头正常；
- 运行 `verify_installation.py --camera`。

### 13.6 MP3 读取失败

先转换为 WAV。推荐 16 kHz、单声道、PCM：

```text
sample rate: 16000 Hz
channels: 1
encoding: PCM 16-bit
```

### 13.7 Excel 批量识别报列名错误

将待识别文本列改名为 `text`、`content`、`文本` 或 `内容`。

### 13.8 第一次预测很慢

文本模型采用延迟加载。第一次预测需要加载大模型，后续通常会明显加快。语音 WavLM ONNX 在 CPU 上也可能需要数秒。

## 15. 最短复现流程

接收者真正需要执行的核心命令只有：

```powershell
git clone https://github.com/Dumbo05/chat_emotion.git
cd chat_emotion
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# 将作者提供的 chat-emotion-runtime-models.zip 放在当前目录

```

如果检查脚本全部显示 `[OK]`，对方即可使用与你电脑上相同的推理功能。
